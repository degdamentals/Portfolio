Linkedln

Plateforme de mise en relation alternants / entreprises

2024 - 2025 Projet Alternance En développement
Voir sur GitHub

Contexte et Objectifs

Contexte du projet

Face aux difficultés rencontrées par les étudiants en recherche d'alternance et les entreprises à la recherche de profils qualifiés, le projet Linkedln vise à créer une plateforme dédiée facilitant la mise en relation directe entre ces deux publics.

Ce projet s'inscrit dans le cadre de mon alternance et répond à un besoin concret observé dans le monde de la formation professionnelle. L'objectif est de simplifier et d'accélérer le processus de recrutement en alternance.

Objectifs principaux

  • Créer une plateforme web responsive permettant aux alternants de créer leur profil et de postuler à des offres
  • Offrir un espace entreprise pour publier des offres d'alternance et consulter les profils des candidats
  • Mettre en place un système de matching intelligent entre les compétences recherchées et les profils disponibles
  • Garantir la sécurité des données avec authentification sécurisée et protection des informations personnelles
  • Faciliter la communication via un système de messagerie intégré entre candidats et recruteurs

Problématiques techniques résolues

  • Conception d'une architecture applicative évolutive et maintenable
  • Mise en place d'une authentification multi-rôles (alternants, entreprises, administrateurs)
  • Gestion optimisée des données utilisateurs et des offres d'emploi
  • Création d'algorithmes de recherche et de matching pertinents
  • Développement d'interfaces utilisateur intuitives et accessibles

Environnement Technique

Frontend (Prévu)

  • React Framework JavaScript pour l'interface utilisateur
  • TypeScript Typage statique pour plus de robustesse
  • Tailwind CSS Framework CSS utilitaire pour le design
  • Axios Client HTTP pour les appels API
  • React Router Gestion de la navigation

Backend (Prévu)

  • Node.js Environnement d'exécution JavaScript côté serveur
  • Express Framework web pour les API REST
  • JWT Authentification par tokens sécurisés
  • Bcrypt Hachage des mots de passe
  • Multer Gestion des uploads de fichiers (CV, photos)

Base de données (Prévue)

  • PostgreSQL SGBD relationnel pour les données structurées
  • Prisma ORM moderne pour Node.js
  • Redis Cache pour améliorer les performances

Outils et DevOps

  • Git / GitHub Gestion de version et hébergement du code
  • VS Code Environnement de développement
  • Postman Tests des API REST
  • Docker Conteneurisation de l'application
  • GitHub Actions CI/CD pour les déploiements automatisés

Difficultés Rencontrées et Améliorations Futures

Difficultés

  • Choix technologiques : Comparaison entre différentes stacks techniques (Next.js vs React, MySQL vs PostgreSQL) pour répondre aux besoins du projet
  • Algorithme de matching : Conception d'un système pertinent pour faire correspondre compétences et offres de manière efficace
  • Sécurité : Mise en place d'une authentification robuste et protection contre les vulnérabilités courantes (OWASP Top 10)
  • Gestion du temps : Équilibre entre l'alternance, les cours et le développement du projet

Améliorations Prévues

  • Messagerie en temps réel : Intégration de WebSockets pour un système de chat instantané entre alternants et entreprises
  • Notifications push : Alertes automatiques pour les nouvelles offres correspondant aux profils
  • Système de recommandation : Amélioration de l'algorithme avec machine learning pour des suggestions plus pertinentes
  • Application mobile : Développement d'une version React Native pour iOS et Android
  • Tableau de bord administrateur : Interface avancée avec statistiques et analytics

Conclusion et Apports

Le projet Linkedln représente un défi technique et professionnel majeur dans mon parcours BTS SIO SLAM. Il m'a permis de mettre en pratique l'ensemble des compétences acquises en formation, de la conception à la réalisation d'une application web complète.

Ce projet m'a particulièrement fait progresser sur :

  • La conception d'architectures applicatives scalables et maintenables
  • Le développement full-stack avec des technologies modernes
  • La sécurité applicative et la protection des données personnelles
  • La gestion de projet et l'organisation du travail en autonomie
  • La documentation technique et la communication autour du projet

Bien que le projet soit encore en cours de développement, les fondations sont solides et la roadmap clairement définie. L'objectif est de livrer une première version fonctionnelle d'ici la fin de l'année scolaire 2024-2025, qui pourra servir de support concret pour les épreuves E4 et E5 du BTS.
